# Product requirements

These documents define a sequence of independently demonstrable features for a browser-based chiptune DAW. They are deliberately ordered so that each completed PRD leaves the product in a usable state.

| PRD | Feature | Delivery tier |
| --- | --- | --- |
| [PRD 01](./01-playable-instrument.md) | Playable instrument | Core |
| [PRD 02](./02-single-track-step-sequencer.md) | Single-track step sequencer | Core |
| [PRD 03](./03-pattern-editing.md) | Pattern editing | Core |
| [PRD 04](./04-multi-track-songs.md) | Multi-track songs | Core |
| [PRD 05](./05-song-arrangement.md) | Song arrangement | Extended |
| [PRD 06](./06-project-persistence.md) | Project persistence | Core |
| [PRD 07](./07-audio-export.md) | Audio export | Extended |
| [PRD 08](./08-reactive-visualiser.md) | Reactive visualiser | Stretch |
| [PRD 09](./09-visualiser-editor.md) | Visualiser editor | Stretch |
| [PRD 10](./10-sharing-and-playback-pages.md) | Sharing and playback pages | Ultimate stretch |

The intended first release boundary is PRDs 01–04 and 06. PRD 05 and PRD 07 deepen the music workflow, while PRDs 08–10 add the audiovisual and sharing experiences.
